修改样式属性

  • 方法① 元素.style.样式名 = ‘样式值’;

    • 注意如果CSS的样式名中含有 -
      • 这种名称在JS中是不合法的,比如background - color
      • 这种样式名需要修改为驼峰命名法, 去掉 -   然后将 - 后的字母大写 backgroundColor

  • 通过 style属性设置的样式都是内联样式

    • 而内联样式有较高的优先级,在通过JS修改的样式往往会立即显示
    • 如果样式中写了 !important,则此时样式会有最高的优先级,即使通过JS也不能覆盖该样式

  • 注意, 通过style属性读取和设置的都是内联样式,无法读取样式表中的样式.

  • 方法② 元素.className = ‘class名’

    • 让我们当前元素的class = ‘class名’,会覆盖原来的class名
    • 如果想要保留原来的class名,可以设为 .className = ‘原来的class名 新class名’;

事件对象

  • function(e)
  • 写到我们监听函数的()里面的就是事件对象,当形参来看.它是系统为我们自动创建的,不需要我们传递参数
  • 事件对象是我们事件的一系列相关数据的集合

    • 键盘事件就包含了键盘的信息,判断用户按下了哪个键
  • 常见属性和方法
    • e.target  —-标准

      • return false 也能阻止默认行为但只限于传统注册方式
    • e.stopPropagation()
      • 阻止冒泡,一般是在孩子调用
      • 兼容性: window.e.cancelBubble = true;

事件的冒泡(Bubble)

  • 指事件的向上传导,当后代元素上的事件被触发时,其祖先元素的相同事件也会被触发

  • (我亲了小女孩了一下,小女孩打了我一顿,她爸爸又打了我一顿,她爷爷又又打了我一顿,只亲了一下却挨了三顿打.)

  • 事件的委派

    • 我们希望只绑定一次事件,即可应用到多个元素上,即使元素是后添加的.
    • 尝试将其绑定给元素的共同的祖先元素
    • 原理:不给每个子节点单独设置事件监听器,而是把事件监听器设置在其父节点上,然后利用冒泡原理设置子节点
      • 给ul注册点击事件,利用事件对象的target来找到当前点击的li,因为点击li,事件会冒泡到ul上,ul有注册事件就会触发事件
    • 指将事件统一绑定给共同的祖先元素**,这样当后代元素上的事件触发时,会一直冒泡到祖先元素,**从而通过祖先元素的响应函数来处理事件
    • 事件委派是利用了冒泡,通过委派可以减少事件绑定的次数,提高程序的性能

代码如上,上午在写JavaScript的tab栏切换小练习时,按照教程来说可以自定义一个class: index 来设置序号,方便切换相对应的区块.

但我当时觉得不用设置这个class属性就可以,利用我们得到的是伪数组这个数据集来找,所以我把那两行自定义代码注释掉试了一试.

但结果并非我想的那样,截图如下

后来就观察我这代码哪里有问题. 通过调试终于发现了问题!

items[i].style.display = ‘block’;

就是这一行代码的i出现了问题,这个i根本不是指的0-而是指的一个具体的数,首选判断这个i并不是指第一个for循环里的i,而是指函数里的for循环里的i,这是一个函数作用域,这点搞明白了接下来往下看,按理说,既然是函数里的i,那最起码这个i应该是一个具体的数值,肯定是一个点击事件是有效果的,但并没有,这个i到底是多少.

回到函数里的for循环,我当初自以为的认为i=4,但并没有出现具体的效果,直到看到了i++,明白到,当i=4时,它还在执行完代码后还会进行i++操作,也就是i=5,然后再去判断i < lis.length从而结束循环.

至此问题解决,这个当初写的时候我真的感觉理论上没有问题,但现实打脸.还是自定义一个新的class属性来做吧,代码截图如下.

while(条件表达式){

语句…

}

  • while语句在执行时,先对条件表达式进行求值判断
    • 如果值为true,则执行一次循环体
      • 循环体执行完毕后,继续对表达式进行判断
        • 如果值位true,则继续执行循环体
        • 如果判断值为false,则终止循环
  • 条件表达式为true时,叫做死循环
    • 慎用!
  • 终止循环
    • 可以使用break;来终止循环
  • do….while
    • 语法:

do{

语句…

}while(条件表达式)

  • 流程
    • do…while语句在执行时,会先执行循环体
      • 循环体执行完毕以后,再对while后的条件表达式进行判断

如果表达式结果为true,则继续执行循环体…..

如果结果为false,则终止

  • 不同于while,do…while 会先执行一次再判断

  • for

    • 提供了专门的位置来放三个表达式
      1. 初始化表达式
      2. 条件表达式
      3. 更新表达式
    • 语法
      • for(初始化表达式;条件表达式;更新表达式){
      • 语句…
      • }

  • 执行流程

    1. 执行初始化表达式**(只会执行一次)**
    2. 执行条件表达式**,**判断是否执行循环
      1. 如果为true**,**则执行循环语句
      2. 如果为false**,**终止循环
    3. 执行更新表达式**,更新完毕回到步骤2**

  • for中的三部分都可以省略,也可以写在外面

    • for( ;; ){

语句…

}

  • 利用字符串来控制在一行
    • var str = ‘’;
    • str = str + xx ;

    • for (外层初始化变量; 外层条件表达式; 外层更新表达式){

for(里层初始化变量; 里层条件表达式; 里层更新表达式){

//执行语句;

}

str = str + ‘\n’;

  • }
  • 可以把里层的循环看作外层循环的语句
  • 外层循环循环一次,里层的循环执行全部

诗歌: 别丢掉

2020-07-10

[hermit autoplay=”false” mode=”auto” preload=”auto” ]netease_playlist#:5105159092[/hermit]

心血来潮写下的,明明刚才在玩,突然脑海里浮现了<朗读者>这个节目,好久了,上了大学就没看过了,记得是在高三那年经常看的,语文课还是班会课?可能都有吧.

然后便打开笔记本打开浏览器去搜”朗读者”这三个字,就是这三个字,然后就看见了.

别丢掉

作者:林徽因

别丢掉

这一把过往的热情

现在流水似的

轻轻

在幽冷的山泉底

在黑夜,在松林

叹息似的渺茫

你仍要保存着那真

一样是明月

一样是隔山灯火

漫天的星,只有人看不见

梦似的挂起

你向黑夜要回

那一句话 —— 你仍得相信

山谷中留着

有那回音!

是高中吧,那时候怎么说呢,自卑无趣成绩也不算太好,现在也差不多,怎么开始接触到林徽因的也忘了,只记得当初拜托不在学校住宿晚上回家的同学在淘宝上买的,然后过了几天就到了.《别丢掉》这首诗也不知怎么就是被她吸引了.有一阵子早上的早自习不是课本,而是这首诗,上了大学也带着那本书来可惜仅仅是带着.

有多久了,两年多了吧,今天在朗读者上又看见了她.当初看这首诗的时候是一个滋味,现在再看一遍又是另一个滋味,大一的时候我有一个小粉本上面抄着各种各样的诗词还有歌还有红楼梦里的片段那个时候可能是到了一个新环境灵感比较多吧.但也仅仅是大一,现在的那个小粉本和书一直放在书桌上,一直一直.

未来肯定是不好过的,不尽人意,也不会想去再看这样的书,毕竟…… 今天是脑海里突然想了起来就写一些.

只希望未来我真的能够”别丢掉”吧.
